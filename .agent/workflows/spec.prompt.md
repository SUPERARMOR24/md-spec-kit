# ゴール
`input/specify.md` を読み、欠落を補いながら「絵文字だけでつながるSNS」の仕様書を `output/spec.md` に日本語で出力する。

# 手順
1. 入力を確認し、前提・不明点を洗い出す。
2. 必要に応じて仮定を置き、根拠を簡潔に記す（仮定リストを末尾に列挙）。
3. `templates/template.spec.md` の構成を踏まえて、実装可能な粒度の仕様を埋める。
4. UIは高校生が操作できるシンプルさを優先し、「絵文字のみ投稿・絵文字リアクション・タイムライン閲覧」が最小機能。
5. 技術スタックは Node.js + React 系を想定してよい。APIやストレージは簡易なモックで可。

# 出力フォーマット
- ファイル: `output/spec.md`
- テンプレートに沿ったセクション構成で記述する。
- すべて日本語。表や箇条書きを活用し、冗長な説明を避ける。

# 制約
- 事実根拠がない情報は「仮定」と明記する。
- 体験授業で65分以内にデモが成立する範囲に絞る。
- 絵文字のみの投稿制約が緩まないよう注意する（本文・コメントとも）。
